🔴 Самый очевидный следующий шаг: именная парадигма
Вся работа с sanskrit_pxn_v4.html — про глаголы. Но cs.csv показал: именных токенов 2.28М против 781k глагольных. Nominative Sg = 34.6% всех именных форм, Dual везде < 1%. Данные для аналогичного инструмента уже есть в корпусе — нужно только собрать парадигмы существительных (-a, -ā, -i, -u, -an, -in, -ant) и прилагательных. Это прямое продолжение в той же логике: ячейки падеж × число, цвет по корпусной частоте, примеры.

Студент не знает, с чего начать. Логичный следующий шаг — landing page: одна страница с картой инструментов, рекомендованным маршрутом (Stage 1 → 2 → 3 → 4), и прямыми ссылками. Это не новые данные — это организация уже сделанного.

🟢 Связка парадигма ↔ конкорданс ↔ пассажи
Сейчас в sanskrit_pxn_v4.html при клике на ячейку открывается панель с примерами из DCS. Но пассажи D3 и конкорданс — отдельные инструменты. Логичная связка: из панели контекстов — переход «найти пассаж с этой формой» → D3 открывает соответствующий текст. Это замыкает цикл: узнал форму → увидел её в корпусе → читаешь реальный текст.

## Что ещё разумно с дидактической точки зрения:
🔴 Высокий приоритет — принципиально новое
1. Корпусная засвидетельствованность конкретной формы конкретного корня
Сейчас цвет ячейки показывает, как часто встречается любой глагол в форме 3sg Perfect. Но сколько раз встречается именно jagāma (√gam 3sg Perfect)? Это совсем другой вопрос. Данные есть в 12.csv/15.csv — можно показать «√gam засвидетельствован в Perfect 3sg: 847 раз», отдельно от общей статистики тенса.
2. Контекстные примеры из корпуса
Клик на ячейку → 2–3 реальные строфы из DCS где стоит именно эта форма этого корня. У нас уже есть конкордансный индекс (conc_part1/2/3.json). Соединить с парадигмой.
3. Визуальное разбиение основа + окончание
karo-ti / bhava-ti / gaccha-ti — раскрасить основу серым, окончание синим. Студент видит, что -ti инвариантно, а различается только основа. Особенно важно при переключении между корнями.

🟡 Средний приоритет
4. Сравнение двух корней
Кнопка «Сравнить» → второй ряд форм под первым в той же ячейке. Например √kṛ vs √bhū: в Perfect видно что kṛ редупликация ca-kāra vs bhū → ba-bhūva — оба типа сразу.
5. Метка глагольного класса с формулой основы
√kṛ (кл. 8, -u-): karo-, kuru- / √bhū (кл. 1, -a-): bhava- / √dā (кл. 3, редупл.): da-dā-, dat-. Одна строка под переключателем корня — студент понимает, почему формы выглядят так, как выглядят.
6. «Что учить сейчас» — персональный маршрут
Ввод: «я знаю Present и Optative». Вывод: «следующие 3 формы дадут +X% охвата — это PPP, Absolutive, Perfect». Фактически это ползунок, но со смысловым объяснением перехода.
7. Режим карточек (flashcard)
Показывает ячейку с вопросом: «√bhū · 3sg · Imperfect → ?», ответ — abhavat/abhavata. Данные уже есть, нужен только интерфейс перелистывания.

🟢 Полезно, но просто
8. Фильтр «только засвидетельствованные»
Скрыть ячейки с прочерком «—» (нулевые формы √as в ātmanepada, пассив √gam в двойственном числе). Таблица становится компактнее и отражает реальность.
9. Печать / PDF одной страницей
Кнопка «Распечатать» → CSS print-media печатает таблицу без управляющих элементов.
